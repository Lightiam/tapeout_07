# LightRail netlist v0.1 — built on the MSI RTX 2070 reference

Per direction: use the MSI GeForce RTX 2070 (MS-V373 r9.1) boardview as the
topology guide and build the custom LightRail architecture on it, toward a
fab-ready file. This documents both the reference and the first-pass netlist.

## 1. The reference — MSI RTX 2070 (GENCAD 1.4 boardview)

Parsed from `msi_reference/msi_rtx2070_netlist.csv` (full extract):

| | |
| --- | ---: |
| Nets | 1,760 |
| Pin connections (nodes) | 11,026 |
| Components | 3,054 |

**Reusable topology patterns** (what a shipping high-power accelerator does):

| MSI pattern | Evidence | LightRail analogue |
| --- | --- | --- |
| Multi-phase core VRM | `NVVDD` (951 pins), `PS_NVVDD_PH1..N`, `SNN_NVVDD_GL#_PH#` | 24-phase VPD → NCE `VDD_CORE_0V8_A/B` |
| Dedicated memory power | `FBVDDQ` (943 pins) | `VDD_HBM_1V2` / `VDD_HBM_VDDQ_1V1` |
| Segmented memory buses | `FBA/FBB/FBC/FBD_D<0..63>`, `_CMD`, `_DBI`, `_EDC` (4 channels) | per-NCE HBM groups (6 stacks each) |
| PCIe diff-pair lanes | `PEX_TX#/RX#(*)` (16) | slot ↔ PEX88096 ↔ NCE lanes |
| Rail hierarchy from 12V | `12V_F` → `5V`, `NV3V3`, `1V8_*` | `+12V_IN` → `VDD_*` rails |
| Massive GND/decoupling | `GND` (3,305 pins) | global `GND` + C1–C16 |

## 2. The build — LightRail netlist v0.1

Generated by `scripts`-style construction from (a) the authoritative pin names
in the Rev 4.0 schematic PDF (`../schematic_capture/component_pin_inventory.csv`)
and (b) the MSI patterns above. Output: `lightrail_netlist.csv` / `.json`,
diagram `lightrail_netlist_architecture.png`.

| | Before (PCB as-found) | **LightRail v0.1** |
| --- | ---: | ---: |
| Production components connected | 3 / 82 | **82 / 82** |
| Nets | 64 (U1↔U3↔U4 only) | **155** |
| Pin connections | 212 | **594** |

**What is defined (engineering-derived, defensible):**
- **Power tree** — `+12V_IN` (from all 4 PCIe slots) → ISL69260 controllers +
  ISL99390 stages → `VDD_CORE_0V8_A` (12 phases, U210–U221) and `_B`
  (12 phases, U222–U233) → NCE `VDD_CORE`; ADP7118 LDOs → `VDD_RF_0V9`/`VDD_PLL_0V9`;
  `VDD_HBM_1V2`, `VDD_IO_1V8` rails. Mirrors MSI NVVDD multi-phase + FBVDDQ.
- **Clocking** — Si5395A → `CLK_REF_100M`, `CLK_CORE_2G`, `CLK_NOC_1G`,
  `HBM_REFCK_A/B`, `PCIE_REFCLK`.
- **TFLN control** — U1→U3 `TX_P/N0..3`, U4→U3 B-side; RF chain U50–U53 driving MZM.
- **PCIe** — each slot J20–J23 ↔ PEX88096 (8 lanes TX/RX); NCE `PCIE_TX` uplinks;
  retimers in path; `PERST_N`/`REFCLK` distributed.
- **Management** — AST2600 BMC I2C/SPI/UART sideband to NCEs and PMBus to VRMs.
- **Decoupling** — C1–C16 across the nearest rail↔GND.
- **Global GND** — all 82 parts referenced.

## 3. What is explicitly NOT yet resolved (needs real device data)

This is a **v0.1 architectural netlist**, not a validated production netlist. It
must not be fabricated as-is. Open items, each requiring the actual device
datasheet / ball-map:

1. **HBM4 DQ ball-map** — the 1024-ball HBM4 ↔ NCE data-bus assignment
   (which NCE ball drives which stack's `DQ`/`DBI`/`EDC`) is defined only at the
   `CK`/`REFCK`/power level here. The PDF footprint is a 16-pin placeholder, not a
   real BGA-1024 land pattern.
2. **NCE BGA-2500 pin-out** — the Rev 4.0 symbol exposes named functional pins but
   not the full 2,500-ball map; core power/ground ball fan-out is nominal.
3. **Full PCIe lane map** — Gen5 lane polarity/ordering and which NCE lanes reach
   which switch/retimer/slot need the PEX88096 and NCE lane tables.
4. **Placeholder MPNs** — the sheet states user-supplied MPNs "require
   authoritative validation" (`COMPONENT_VALIDATION.md`, not in repo).
5. **Target board** — 4-layer (295×140) vs 22-layer (305×280) still unresolved
   (see `../schematic_capture/SCHEMATIC_SOURCE_FINDING.md`). The **logical netlist
   here is layer-independent**, so this does not block netlist work, but it blocks
   stackup/routing.

## 4. Path to a fab-ready file from here

1. Resolve items 1–4 against real datasheets (the `datasheets`/`digikey` skills can
   pull them) → promote v0.1 to a pin-complete netlist.
2. Capture it as a KiCad schematic (hierarchical, per the 7 subsystems) → ERC clean.
3. Back-annotate to the PCB → drive schematic-parity to 0.
4. Choose the layer target, define the impedance stackup, route + pour planes.
5. Regenerate Gerbers/drill/IPC-netlist, re-run DRC/DFM/DFA → fab-ready package.

v0.1 delivers step 0→1's foundation: the board now has a complete, documented,
82/82-connected logical netlist for the first time, grounded in a real
shipping-board reference.
